# /meeting-agent - 실제 멀티에이전트 팀 회의

## 개요
실제 Agent 도구를 사용해 역할별 독립 에이전트를 병렬 실행하는 팀 회의.
`/meeting-multi`와 달리 각 에이전트가 완전히 독립된 컨텍스트에서 실행됨.

---

## Step 1: 회의 설정 확인

AskUserQuestion 도구로 다음 2가지를 물어보세요:

**질문 1 - 회의 주제:**
- $ARGUMENTS 가 있으면 그걸 주제로 사용
- 비어있으면 주제 입력 요청

**질문 2 - 역할 구성:**
기본값: `콘텐츠기획x1, 성장기획x1, 백엔드x1, 프론트엔드x1, QA x1, UI x1, UX x1, PD x1` (총 8명)
사용자가 변경하고 싶으면 자유 입력 가능.
형식 예시: `백엔드x1, 프론트x1, QA x1, PD x1`

---

## Step 2: 에이전트 병렬 실행

확인된 역할 구성으로 **Agent 도구를 동시에(한 번에) 실행**하세요.

각 에이전트 프롬프트 구조:
```
당신은 [역할명] #[번호] 입니다.
회의 주제: [주제]

[역할]의 전문적 관점에서 다음을 작성하세요:
1. 핵심 의견 (3줄 이내)
2. 우려사항 또는 리스크
3. 제안하는 방향

다른 역할의 관점은 고려하지 말고, 본인 역할 관점만 집중해서 작성하세요.
```

역할별 포커스 가이드:
- **콘텐츠기획**: CCA-F 콘텐츠 구조, 학습 경로, MDX 분류, 수험생 니즈
- **성장기획**: SEO, 전환 퍼널, KPI, 경쟁 포지셔닝
- **백엔드**: Supabase, Prisma, Server Actions, RLS
- **프론트엔드**: Next.js App Router, MDX, Tailwind, 다국어
- **QA**: 빌드 검증, Lighthouse, 접근성, 크로스브라우저
- **UI**: Stone+Indigo 테마, CSS 변수, 다크모드, 컴포넌트
- **UX**: 학습 UX, 수험생 여정, WCAG, 정보구조
- **PD**: Phase 범위 관리, 일정, Go/No-Go, 스코프 크립 방지

---

## Step 3: 결과 종합

모든 에이전트 결과를 받은 후:

```
## 🏁 팀 회의 결과: [주제]

### 역할별 핵심 의견
[각 에이전트 요약]

### 공통 합의점
[여러 역할이 동의하는 부분]

### 주요 쟁점
[역할 간 의견 차이가 있는 부분]

### 권장 다음 액션
[구체적 실행 항목 3가지 이내]
```
