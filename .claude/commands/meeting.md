# /meeting - 1인 순차 팀 회의 (경량 버전)

Topic: $ARGUMENTS (default: "현재 단계에서 추가할 기능")

## 지시사항

CCAFup 프로젝트의 8인 전문가 관점을 **1명이 순차적으로 분석**한다.
Task 에이전트 호출 없이 직접 수행하는 경량 버전.

---

## 8인 전문가 관점

### 기획팀
- **A1 콘텐츠 전략 (현우)**: CCA-F 콘텐츠 구조, 학습 경로, MDX 분류, 수험생 니즈
- **A2 성장 전략 (소연)**: SEO, 전환 퍼널, KPI, 경쟁 포지셔닝, Phase 분류

### 개발팀
- **B1 백엔드 (태준)**: Supabase, Prisma, Server Actions, RLS, DB 스키마
- **B2 프론트엔드 (미래)**: Next.js App Router, MDX 렌더링, Tailwind, 다국어

### 디자인팀
- **C UI 디자이너 (다은)**: Stone+Indigo 테마, CSS 변수, 다크모드, 컴포넌트
- **D UX 디자이너 (승호)**: 학습 UX, 수험생 여정, 접근성(WCAG), 정보구조

### 품질팀
- **E QA (윤서)**: 빌드 검증, Lighthouse, 접근성, 크로스브라우저
- **F PD (지민)**: Phase 범위 관리, 일정, Go/No-Go, 스코프 크립 방지

```text

- 기술: Next.js 15 + Supabase + Prisma + MDX + Tailwind CSS

- 주요 파일/문서:
  - docs/01_마스터목차.md (기능 목록)
  - docs/02_디자인시스템.md
  - docs/03_테마시스템.md

  - docs/05_Phase분리.md
```

## 1단계: 현황 파악

- CLAUDE.md, docs/ 디렉토리 읽고 현재 상태 확인
- src/ 디렉토리 구조 파악
- 현황 요약 작성

---

## 2단계: 8인 관점 순차 분석

Task 에이전트 호출 없이 직접 8개 관점에서 분석한다.
각 관점에서 2-3개 제안.

---

## 3단계: 합의 도출

- 여러 관점에서 긍정 평가 → 우선순위 상승
- 관점 간 충돌 → 명시적 기록
- Phase 1/2/3 분류 명확히
- 최종 판정: 채택/보류/기각

---

## 4단계: 회의록 저장

`docs/meeting/plan/single/YYYY-MM-DD-HHmm-{topic-summary}.md`에 저장.

---

## 5단계: 구현문서(impl) 생성

채택 항목을 **영문 구현문서**로 분리 생성.
**파일명**: `docs/meeting/impl/{회의록 파일명}-impl.md`

---

## 6단계: 자동 커밋 & 푸시

회의록과 impl 문서 생성 완료 후 `/summitdocs` 실행.
