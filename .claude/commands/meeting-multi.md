# /meeting-multi - 역할극 기반 팀 회의 (8인)

## 개요
Claude 1개가 여러 역할을 순차적으로 연기하는 역할극 방식.
컨텍스트를 공유하므로 앞 발언이 뒤 역할에 영향을 줌.

기본 구성: 콘텐츠기획x1, 성장기획x1, 백엔드x1, 프론트엔드x1, QA x1, UI x1, UX x1, PD x1 (총 8명)

---

## Step 1: 회의 설정 확인

AskUserQuestion 도구로 다음을 확인하세요:

**질문 1 - 회의 주제:**
- $ARGUMENTS 가 있으면 그걸 주제로 사용
- 비어있으면 주제 입력 요청

**질문 2 - 역할 구성:**
기본값: `콘텐츠기획x1, 성장기획x1, 백엔드x1, 프론트엔드x1, QA x1, UI x1, UX x1, PD x1` (총 8명)
변경하고 싶으면 자유 입력.

---

## Step 2: 역할극 순차 실행

각 역할을 **순서대로** 연기하세요. 매 역할마다 이전 발언을 읽고 답변.

각 역할 발언 형식:
```
---
**[역할명]**
[핵심 의견 — 3줄 이내]
[우려사항 또는 리스크]
[제안 방향]
---
```

역할별 포커스:
- **콘텐츠기획**: CCA-F 콘텐츠 구조, 학습 경로, MDX 분류
- **성장기획**: SEO, 전환 퍼널, KPI, 경쟁 포지셔닝
- **백엔드**: Supabase, Prisma, Server Actions, RLS
- **프론트엔드**: Next.js App Router, MDX, Tailwind, 다국어
- **QA**: 빌드 검증, Lighthouse, 접근성, 크로스브라우저
- **UI**: Stone+Indigo 테마, CSS 변수, 다크모드
- **UX**: 학습 UX, 수험생 여정, WCAG, 정보구조
- **PD**: Phase 범위 관리, 일정, Go/No-Go
