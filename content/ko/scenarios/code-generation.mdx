## 시나리오 개요

개발팀이 Claude Code를 활용하여 코드 생성, 리뷰, 리팩토링 워크플로우를 자동화하는 시스템을 구축합니다. CLAUDE.md 파일을 통해 프로젝트 컨텍스트를 제공하고, 슬래시 커맨드로 반복 작업을 표준화합니다.

## 관련 도메인

- **Claude Code Configuration & Workflows** (20%): CLAUDE.md 설정, 슬래시 커맨드
- **Prompt Engineering** (20%): 코드 생성 프롬프트 설계
- **Agentic Architecture** (27%): 에이전트 루프 기반 코드 생성
- **Context Management** (15%): 프로젝트 컨텍스트 유지

## 워크스루

### Step 1: CLAUDE.md 설정

프로젝트의 컨벤션과 규칙을 CLAUDE.md에 정의하여 Claude Code가 일관된 코드를 생성하도록 합니다.

```markdown
# Project Rules
- TypeScript strict mode, any 금지
- 파일명: kebab-case.ts
- 컴포넌트: PascalCase export
- 한 파일 한 컴포넌트
```

<Tip title="설계 포인트">
CLAUDE.md는 프로젝트 루트, 서브디렉토리, 사용자 홈에 배치할 수 있습니다. 우선순위는 가장 구체적인 경로가 높습니다. 팀 전체 규칙은 루트에, 모듈별 규칙은 해당 디렉토리에 배치하세요.
</Tip>

### Step 2: 슬래시 커맨드로 워크플로우 표준화

반복적인 코드 생성 작업을 슬래시 커맨드로 정의합니다.

| 커맨드 | 용도 | 예시 |
|--------|------|------|
| `/generate` | 새 컴포넌트 생성 | `/generate Button with variants` |
| `/test` | 테스트 파일 생성 | `/test for utils/format.ts` |
| `/review` | 코드 리뷰 실행 | `/review --focus security` |
| `/refactor` | 리팩토링 제안 | `/refactor extract-method` |

### Step 3: 코드 생성 흐름

Claude Code의 코드 생성은 에이전트 루프로 동작합니다.

1. **컨텍스트 수집**: CLAUDE.md + 관련 파일 읽기
2. **계획 수립**: 생성할 코드의 구조 결정
3. **코드 작성**: 컨벤션에 맞는 코드 생성
4. **검증**: 린터, 타입 체커 실행으로 품질 확인
5. **수정**: 오류 발견 시 자동 수정 루프

<Warning title="함정 문제">
Claude Code가 생성한 코드를 검증 없이 바로 커밋하는 것은 안티패턴입니다. 반드시 린터와 테스트를 통한 검증 단계를 포함해야 합니다. 시험에서 "자동 생성된 코드의 품질을 보장하는 방법"을 묻는 문제가 출제됩니다.
</Warning>

### Step 4: Git 통합

Claude Code는 Git 워크플로우와 긴밀하게 통합됩니다.

- **커밋 메시지 자동 생성**: Conventional Commits 형식 준수
- **PR 설명 자동 작성**: 변경 사항 요약 + 테스트 계획
- **코드 리뷰 자동화**: CI/CD 파이프라인에서 Claude Code 실행
