## 시나리오 개요

CI/CD 파이프라인에 Claude Code를 통합하여 코드 품질 검사, 테스트 자동 생성, 배포 전 검증을 자동화합니다. 헤드리스 모드로 실행되는 Claude Code가 파이프라인의 각 단계에서 지능적인 검증을 수행합니다.

## 관련 도메인

- **Claude Code Configuration & Workflows** (20%): CI/CD 통합, 헤드리스 모드
- **Agentic Architecture** (27%): 파이프라인 에이전트 설계
- **Tool Design & MCP** (18%): CI/CD 도구와의 연동
- **Context Management** (15%): 파이프라인 컨텍스트 유지

## 워크스루

### Step 1: CI 파이프라인 통합

Claude Code를 GitHub Actions에 통합합니다.

```yaml
# .github/workflows/claude-review.yml
name: Claude Code Review
on: [pull_request]
jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Claude Code Review
        run: |
          claude --print "Review this PR for security issues,
          performance problems, and coding convention violations.
          Output as structured JSON."
```

<Tip title="설계 포인트">
CI 환경에서 Claude Code는 `--print` 플래그로 비대화형(non-interactive) 모드로 실행합니다. 출력은 JSON 형식으로 구조화하여 후속 파이프라인 단계에서 파싱할 수 있게 합니다.
</Tip>

### Step 2: 자동 테스트 생성

새 코드가 추가될 때 자동으로 테스트를 생성합니다.

| 단계 | 동작 | 실패 시 |
|------|------|---------|
| 커버리지 분석 | 새 코드의 테스트 커버리지 확인 | 테스트 생성 트리거 |
| 테스트 생성 | 단위/통합 테스트 자동 생성 | PR에 코멘트 |
| 테스트 실행 | 생성된 테스트 실행 및 검증 | 수정 루프 실행 |
| 결과 보고 | 커버리지 변화 리포트 | PR 차단 |

### Step 3: 배포 전 검증

프로덕션 배포 전 최종 검증을 수행합니다.

- **Breaking Change 감지**: API 변경이 하위 호환성을 깨는지 확인
- **환경 변수 검증**: 필요한 환경 변수가 모두 설정되어 있는지 확인
- **마이그레이션 검사**: DB 스키마 변경의 안전성 검증
- **보안 스캔**: 하드코딩된 비밀키, 취약한 의존성 확인

### Step 4: 롤백 판단 자동화

배포 후 모니터링 지표를 분석하여 롤백 필요성을 판단합니다.

```
배포 → 5분 대기 → 에러율 확인 → 정상이면 완료
                              → 이상이면 자동 롤백 + 알림
```

<Warning title="함정 문제">
CI/CD 파이프라인에서 Claude Code의 판단만으로 프로덕션 배포를 자동 승인하는 것은 안티패턴입니다. 최종 배포 승인은 반드시 Human-in-the-Loop로 구성해야 합니다. 시험에서 "자동화해도 되는 단계 vs 사람의 승인이 필요한 단계"를 구분하는 문제가 출제됩니다.
</Warning>

## 출제 포인트

4. Human-in-the-Loop가 필수인 파이프라인 단계
5. 롤백 판단의 자동화 기준과 한계
