## 시나리오 개요

개발 팀이 Claude를 일상 워크플로우에 통합하여 코드 리뷰, 디버깅, 리팩토링, 문서화 등의 반복 작업을 자동화합니다. IDE 확장, CLI 도구, CI/CD 파이프라인을 통해 개발자 생산성을 극대화하는 방법을 설계합니다.

## 관련 도메인

- **Claude Code Configuration & Workflows** (20%): CLI 도구 활용, 워크플로우 자동화
- **Prompt Engineering** (20%): 개발 작업별 프롬프트 설계
- **Agentic Architecture** (27%): 개발 워크플로우 에이전트 설계
- **Tool Design & MCP** (18%): IDE 통합, 개발 도구 연동

## 워크스루

### Step 1: 개발 환경 통합

Claude를 개발 환경에 통합하는 3가지 접점을 설정합니다.

| 접점 | 도구 | 사용 사례 |
|------|------|----------|
| IDE | VS Code / JetBrains 확장 | 인라인 코드 제안, 리팩토링 |
| CLI | Claude Code | 복잡한 코드 생성, 프로젝트 수정 |
| CI/CD | GitHub Actions | 자동 코드 리뷰, 테스트 생성 |

### Step 2: 코드 리뷰 자동화

Pull Request가 생성되면 자동으로 코드 리뷰를 수행합니다.

1. **변경사항 분석**: diff 파싱, 영향 범위 파악
2. **규칙 기반 검사**: 코딩 컨벤션, 보안 취약점, 성능 이슈
3. **맥락 기반 리뷰**: 비즈니스 로직 정합성, 설계 패턴 일관성
4. **피드백 작성**: 구체적인 개선 제안 + 코드 예시

<Tip title="설계 포인트">
코드 리뷰 에이전트는 "차단(blocking)"과 "제안(suggestion)"을 구분해야 합니다. 보안 취약점은 차단, 스타일 개선은 제안으로 분류하세요. 시험에서 리뷰 피드백의 우선순위 결정 방법을 묻습니다.
</Tip>

### Step 3: 디버깅 지원

오류 발생 시 Claude가 디버깅을 지원합니다.

- **에러 로그 분석**: 스택 트레이스 파싱, 근본 원인 추정
- **관련 코드 검색**: 오류 발생 지점 주변 코드 분석
- **수정 제안**: 구체적인 코드 변경 사항 제시
- **테스트 케이스**: 버그 재현 및 수정 검증용 테스트 생성

### Step 4: 문서 자동 생성

코드 변경에 따라 문서를 자동으로 업데이트합니다.

| 문서 유형 | 자동화 방법 | 트리거 |
|----------|-----------|--------|
| API 문서 | 코드 주석에서 추출 | API 파일 변경 시 |
| README | 기능 변경 반영 | 신규 기능 PR 머지 시 |
| 변경 로그 | 커밋에서 자동 생성 | 릴리스 시 |

<Warning title="함정 문제">
Claude를 활용한 코드 생성에서 가장 큰 리스크는 "환각(hallucination)"입니다. 존재하지 않는 API나 라이브러리를 생성할 수 있으므로, 항상 컴파일/빌드 검증 단계를 포함해야 합니다.
</Warning>
