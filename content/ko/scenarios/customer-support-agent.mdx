## 시나리오 개요

고객 지원팀이 일일 수백 건의 문의를 처리해야 하는 상황에서, AI 에이전트를 활용하여 문의를 자동 분류하고 1차 해결을 수행하는 시스템을 설계합니다.

## 관련 도메인

- **Agentic Architecture** (27%): 에이전트 루프 설계, 라우팅 패턴
- **Tool Design & MCP** (18%): 외부 시스템(CRM, KB) 연동
- **Prompt Engineering** (20%): 분류 프롬프트, 응답 생성
- **Context Management** (15%): 대화 이력 관리

## 워크스루

### Step 1: 문의 분류 에이전트

고객 문의를 카테고리(결제, 기술, 일반)로 분류하는 에이전트를 설계합니다.

<Tip title="설계 포인트">
분류 정확도를 높이려면 Few-shot 예시를 카테고리별 2-3개씩 포함하세요. 모호한 문의는 "기타" 카테고리로 분류하고 사람에게 에스컬레이션합니다.
</Tip>

### Step 2: 지식 베이스 검색

분류된 카테고리에 따라 관련 FAQ, 매뉴얼을 검색하여 답변 초안을 생성합니다.

- RAG(Retrieval-Augmented Generation)로 관련 문서 검색
- 검색 결과를 컨텍스트에 포함하여 답변 생성
- 검색 결과가 없으면 일반적인 안내 메시지 반환

### Step 3: 응답 생성 및 검증

생성된 답변이 정확한지 검증하는 guardrails를 적용합니다.

| 검증 항목 | 방법 | 실패 시 액션 |
|----------|------|------------|
| 사실 정확성 | KB 문서와 교차 검증 | 사람 에스컬레이션 |
| 톤/매너 | 시스템 프롬프트 규칙 | 자동 수정 |
| 민감 정보 | 정규식 필터 | 마스킹 처리 |

<Warning title="함정 문제">
시험에서 "에이전트가 확신이 없을 때 어떻게 해야 하는가?"라는 유형의 문제가 자주 출제됩니다. 정답은 항상 "사람에게 에스컬레이션"입니다. 에이전트가 추측으로 답변하는 것은 최악의 선택입니다.
</Warning>

## 출제 포인트

1. 에이전트 라우팅 전략에서 fallback 처리 방법
2. RAG 검색 결과가 없을 때의 대응 전략
3. Human-in-the-Loop 에스컬레이션 기준
4. 응답 품질 guardrails의 종류와 적용 순서
