## 도메인별 핵심 요약

### 1. Agentic Architecture & Orchestration (27%)

| 핵심 개념 | 요약 |
|----------|------|
| 에이전트 루프 | Observe → Think → Act + 종료 조건 필수 |
| 오케스트레이션 | Sequential / Parallel / Routing |
| 멀티에이전트 | Supervisor(hub-spoke) vs Peer-to-Peer |
| Guardrails | 입력/출력 검증 + 행동 제한 + Human-in-the-Loop |
| 종료 조건 | 최대 반복 횟수, 목표 달성, 타임아웃 |

**기억할 규칙**: 에이전트가 확신 없으면 → 사람에게 에스컬레이션. 추측 답변은 최악의 선택.

### 2. Claude Code Configuration & Workflows (20%)

| 핵심 개념 | 요약 |
|----------|------|
| CLAUDE.md | 프로젝트 규칙 정의 파일, 계층적 배치 가능 |
| 슬래시 커맨드 | 반복 작업 표준화 (/generate, /test, /review) |
| CI/CD 통합 | `--print` 비대화형 모드, JSON 출력 |
| Git 통합 | Conventional Commits, PR 자동 작성 |
| 검증 루프 | 생성 → 린트 → 테스트 → 수정 → 반복 |

**기억할 규칙**: CLAUDE.md 우선순위 = 가장 구체적인 경로가 높음.

### 3. Prompt Engineering & Structured Output (20%)

| 핵심 개념 | 요약 |
|----------|------|
| JSON Schema | 출력 형식 제어, enum으로 값 제한 |
| Few-shot | 2-3개 예시로 정확도 향상 |
| System Prompt | 역할, 톤, 제약 조건 정의 |
| Chain-of-Thought | 복잡한 추론에 단계적 사고 유도 |
| 검증 루프 | 스키마 검증(프로그래매틱) + 의미 검증(LLM) |

**기억할 규칙**: JSON 모드 ≠ 정확성 보장. 스키마 + 의미 검증 모두 필수.

### 4. Tool Design & MCP Integration (18%)

| 핵심 개념 | 요약 |
|----------|------|
| MCP | Model Context Protocol — 모델과 외부 도구 연결 |
| 툴 경계 | 하나의 도구 = 하나의 책임 (SRP) |
| 에러 핸들링 | 도구 실패 시 graceful fallback |
| 보안 | 도구 권한 최소화, 입력 검증 |
| 설계 원칙 | 명확한 이름, 상세한 설명, 예시 포함 |

**기억할 규칙**: 도구가 너무 많으면 선택 오류 증가. 5-10개가 적정.

### 5. Context Management & Reliability (15%)

| 핵심 개념 | 요약 |
|----------|------|
| 컨텍스트 윈도우 | 토큰 한계 — 효율적 사용 필수 |
| 핸드오프 | 에이전트 간 컨텍스트 전달 구조화 |
| 대화 이력 | 관련 이력만 선별하여 포함 |
| 신뢰도 | 불확실한 정보 명시, 출처 표시 |
| 청킹 | 대용량 문서 → 논리적 단위 분할 |

**기억할 규칙**: 컨텍스트에 모든 것을 넣지 마라. 관련성 높은 정보만 선별.

## Programmatic vs Prompt-based Enforcement

| 항목 | Programmatic | Prompt-based |
|------|-------------|-------------|
| 정의 | 코드로 강제하는 규칙 | 프롬프트로 요청하는 규칙 |
| 신뢰도 | **높음** — 규칙 위반 불가 | 낮음 — 무시될 수 있음 |
| 예시 | JSON Schema 검증, 정규식 필터 | "반드시 JSON으로 응답하세요" |
| 적용 대상 | 형식, 구조, 값 범위 | 톤, 스타일, 추론 방법 |
| 비용 | 구현 비용 발생 | 프롬프트 수정만으로 가능 |
| 결론 | 형식/보안은 반드시 프로그래매틱 | 행동/스타일은 프롬프트 |

## 시험 직전 10분 — 반드시 기억할 규칙

1. **에이전트 실패 시**: 사람에게 에스컬레이션 (추측 답변 금지)
2. **멀티에이전트 실패**: Graceful degradation (전체 중단 금지)
3. **검증**: 프로그래매틱 + LLM 기반 이중 검증
4. **CLAUDE.md**: 구체적 경로 > 일반 경로 우선순위
5. **도구 설계**: 하나의 도구 = 하나의 책임
6. **컨텍스트**: 관련성 높은 정보만 선별 포함
7. **보안**: 도구 권한 최소화, 입력 항상 검증
8. **배포 승인**: 자동화 가능하지만 최종 승인은 사람
9. **JSON Schema**: enum으로 값 제한, required로 필수 필드 명시
10. **시나리오**: 질문을 먼저 읽고, 시나리오에서 핵심만 파악
