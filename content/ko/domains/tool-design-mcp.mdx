## 개요

Tool Design & MCP Integration은 CCA-F 시험의 **18%** 비중을 차지합니다. AI 모델이 외부 도구와 상호작용하는 방법, MCP 프로토콜의 이해, 안전한 도구 설계를 다룹니다.

## 핵심 개념

### 도구 스키마 설계 원칙

AI 모델이 효과적으로 사용할 수 있는 도구를 설계하는 원칙입니다.

- **명확한 이름**: 도구의 목적을 즉시 파악할 수 있는 이름
- **상세한 설명**: 도구의 기능, 입력, 출력을 구체적으로 설명
- **타입 안전성**: 입력 파라미터의 타입과 제약 조건을 명확히 정의
- **에러 처리**: 실패 시 유용한 에러 메시지 반환

<Tip title="시험 포인트">
도구 이름과 설명이 모호하면 모델이 잘못된 도구를 선택하는 원인이 됩니다. "도구를 사용자가 아닌 모델이 사용한다"는 관점에서 설계하세요.
</Tip>

### MCP (Model Context Protocol)

MCP는 AI 모델과 외부 데이터 소스/도구 간의 표준 통신 프로토콜입니다.

- **Server**: 도구와 리소스를 제공하는 측
- **Client**: 도구를 사용하는 AI 애플리케이션 측
- **Transport**: stdio, HTTP 등 통신 계층
- **Resources**: 파일, 데이터베이스 등 읽기 전용 데이터
- **Tools**: 실행 가능한 함수/작업

### 외부 API 연동 패턴

| 패턴 | 설명 | 보안 고려 |
|------|------|----------|
| Direct API Call | 도구가 직접 외부 API 호출 | API 키 관리 필수 |
| Proxy Pattern | 중간 서버를 통해 간접 호출 | 요청 필터링 가능 |
| Webhook | 이벤트 기반 비동기 처리 | 인증 토큰 검증 |

### 도구 오류 처리

- **Graceful Degradation**: 도구 실패 시 대안 경로 제공
- **Retry Logic**: 일시적 오류에 대한 재시도 전략
- **Timeout**: 도구 실행 시간 제한 설정
