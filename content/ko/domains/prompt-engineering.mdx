## 개요

Prompt Engineering은 CCA-F 시험의 **20%** 비중을 차지합니다. 효과적인 프롬프트 작성, 구조화된 출력 생성, 프롬프트 평가 방법론을 다룹니다.

## 핵심 개념

### 시스템 프롬프트 설계

시스템 프롬프트는 모델의 행동 양식을 정의하는 가장 중요한 요소입니다.

- **역할 정의**: 모델이 수행할 역할을 명확히 지정
- **제약 조건**: 하지 말아야 할 행동을 명시
- **출력 형식**: 응답의 구조와 포맷을 지정
- **컨텍스트**: 필요한 배경 정보 제공

<Callout type="info" title="Best Practice">
시스템 프롬프트는 "무엇을 해야 하는지"보다 "왜 이것을 해야 하는지"를 포함할 때 더 효과적입니다.
</Callout>

### 구조화된 출력

JSON, XML 등 정형화된 포맷으로 출력을 생성하는 기법입니다.

- **JSON 모드**: 유효한 JSON으로 응답을 강제
- **XML 태그**: 구조화된 데이터를 XML로 표현
- **스키마 정의**: 출력 스키마를 사전에 정의하여 일관성 보장

### 프롬프트 기법

| 기법 | 설명 | 장점 |
|------|------|------|
| Zero-shot | 예시 없이 지시만으로 | 간단, 빠른 |
| Few-shot | 2-5개 예시 제공 | 일관된 출력 |
| Chain-of-Thought | 단계별 추론 유도 | 복잡한 문제 해결 |
| Self-consistency | 다중 추론 경로 비교 | 정확도 향상 |

### 프롬프트 테스트 및 평가

프롬프트의 성능을 체계적으로 측정하는 방법입니다.

- **A/B 테스트**: 두 프롬프트의 성능 비교
- **Eval 프레임워크**: 자동화된 프롬프트 평가 파이프라인
- **메트릭**: 정확도, 일관성, 지연 시간, 비용 효율성
